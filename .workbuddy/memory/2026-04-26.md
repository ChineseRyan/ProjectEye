# 2026-04-26 工作记录

## ProjectEye 崩溃 Bug 修复

用户发来 ERROR_2026_04_26.log，分析后定位两个关联 bug：

### 根因
`StatisticContext` (EF6 DbContext + SQLite) 未禁用 EF 默认的 `CreateDatabaseIfNotExists` 初始化器。SQLite 提供商不支持 `CreateDatabase`，当数据库文件不存在时（首次运行/被删除），EF 尝试自动建库抛出 `ProviderIncompatibleException`，导致所有数据库操作（FindCreate→Add→StatisticUseEyeData）连锁失败，`todayStatistic` 始终为 null，引发 `NullReferenceException`。

### 修改文件
1. **`Database/StatisticContext.cs`**：
   - 添加 `System.Data.Entity.Database.SetInitializer<StatisticContext>(null)` 禁用 EF 自动建库（注意：必须用完全限定名调用，因为 `Database` 与 `DbContext.Database` 实例属性同名，直接写 `Database.SetInitializer` 会导致 CS0176 编译错误）
   - 新增 `EnsureDatabaseExists()` 方法：首次运行时自动创建 Data 目录和空的 SQLite 数据库文件（`SQLiteConnection.CreateFile`）
   - 添加 `using System;` 引入 `AppDomain`
   - 表结构仍由已有的 `SQLiteBuilder` 手动管理

2. **`Core/Service/StatisticService.cs`**：
   - `Add()` 方法：为 `FindCreate()` 添加 try-catch 保护，catch 后安全 return；增加 `todayStatistic == null` 的双重检查
   - `StatisticUseEyeData()` 方法：为整个业务逻辑添加 try-catch，防止定时器回调崩溃
   - `Save()` 方法：在 `FindCreate()` 后增加 `todayStatistic == null` 的空检查
